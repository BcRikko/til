[Element#innerHTML](https://developer.mozilla.org/ja/docs/Web/API/Element/innerHTML)は、文字列をHTMLとして解釈してDOMに反映するプロパティで、ユーザー入力などをそのまま反映させるとクロスサイトスクリプティング（XSS）攻撃のリスクがある。テキストノードであれば[Node#textContent](https://developer.mozilla.org/en-US/docs/Web/API/Node/textContent)のように安全に反映できるプロパティができたが、いまだにHTMLとしてそのまま使いたいユースケースもある。

そこでFirefoxが先行して[Element#setHTML()](https://developer.mozilla.org/en-US/docs/Web/API/Element/setHTML)という安全に文字列をDOMに反映するための新しいメソッドを実装した。

<Baseline featureId={"sanitizer"} />

## setHTMLUnsafe()とsetHTML()の違い

`setHTML()`に似たようなメソッドで、全主要ブラウザで利用できる[Element#setHTMLUnsafe()](https://developer.mozilla.org/en-US/docs/Web/API/Element/setHTMLUnsafe)というメソッドがある。

`setHTMLUnsafe()`は、ほぼ`Element#innerHTML`と同じで文字列をそのままDOMに反映するため、依然としてXXS攻撃のリスクがある。一方、`setHTML()`は、文字列を無害化してからDOMに反映するため、XXS対策ができる。

```js
const xxs = "<a href='javascript:alert(\"XSS!!\");'>クリックしてください。</a>"
const safeLink = "<a href='https://example.com'>クリックしてください。</a>"


// <a href='javascript:alert("XSS!!");'>クリックしてください。</a>
unsafeElement.setHTMLUnsafe(xxs)

// クリックしてください。
safeElement.setHTML(xxs)
// <a href='https://example.com'>クリックしてください。</a>
safeElement.setHTML(safeLink)
```


## デモ

以下のテキストをコピーして、入力欄に貼り付けてください。

```html
<!-- XXS -->
<a href="javascript:alert('XSS!!');">クリックしてください。</a>

<!-- Safe -->
<a href="https://example.com">クリックしてください。</a>
```

  <section id="demo-innerHTML">

        <input type="text" id="name1" name="name" />

<script>{`
{
  const demo = document.querySelector('#demo-innerHTML')
  const input = demo.querySelector('#name1')
  const output = demo.querySelector('#result1')

      output.textContent = '名前を入力してください。'

  })
