ブラウザがCSSの特定の機能をサポートしているか確認するために[@supports](https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/At-rules/@supports)が提供されている。
`@supports`はfeature queries（機能クエリー）と呼ばれ、[プログレッシブエンハンスメント](https://developer.mozilla.org/en-US/docs/Glossary/Progressive_Enhancement)が実現できる。

<Baseline featureId={"supports"} />

プロパティごとにチェックする場合は、`if(supports(...), ok, ng)` のような対応も可能。

参考: [CSSでif\(\)が使えるようになる](https://bcrikko.github.io/til/posts/2024-07-12/css-if/)


## 単一のプロパティとその値をテストする場合
```css
@supports (accent-color: red) {
  input[type="checkbox"] {
    accent-color: red;
  }
}

@supports not (accent-color: red) {
  input[type="checkbox"] {
    appearance: none;

    &:checked::before {
      background-color: red;
      color: white;
      content: "\2713";
    }
  }
}
```


## 複数のプロパティをテストする場合
```css
@supports (display: grid) and (not (display: subgrid)) {
  /* ... */
}

@supports (display: grid) or (display: flex) {
  /* ... */
}
```

## セレクターをテストする場合

```css
@supports selector(col.selected||td) {
  /* ... */
}
```
